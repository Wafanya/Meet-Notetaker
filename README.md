# Meet Notetaker

Chrome-екстеншн, який автоматично зберігає транскрипти Google Meet у вашу базу даних Notion — з учасниками, тривалістю зустрічі та таймкодами для кожної репліки.

## ✨ Можливості

- 📝 Зберігає транскрипт зустрічі з Google Meet прямо в Notion
- 👥 Автоматично додає список учасників і тривалість
- ⏱ Кожна репліка має таймкод і ім'я мовця
- 🔄 Оновлює існуючу сторінку при повторному збереженні в межах однієї зустрічі
- 📌 Створює окрему сторінку для кожної унікальної зустрічі

## 📦 Встановлення

### 1. Завантаж код

```bash
git clone https://github.com/<your-username>/meet-notetaker.git
```

Або скачай ZIP з GitHub та розпакуй.

### 2. Завантаж екстеншн у Chrome

1. Відкрий Chrome і перейди за адресою `chrome://extensions/`
2. Увімкни **Developer mode** (перемикач у верхньому правому куті)
3. Натисни **Load unpacked**
4. Обери папку з кодом екстеншну
5. Іконка екстеншну з'явиться у панелі браузера

## ⚙️ Налаштування Notion

Перед використанням потрібно отримати **Notion Token** і створити базу даних.

### 1. Створи Internal Integration у Notion

1. Перейди на [notion.so/profile/integrations](https://www.notion.so/profile/integrations)
2. Натисни **+ New integration**
3. Введи назву (наприклад, `Meet Notetaker`) і обери Workspace
4. Натисни **Save**
5. Скопіюй **Internal Integration Token** — він починається з `ntn_...` (або `secret_...` для старих інтеграцій)

### 2. Створи базу даних у Notion

Створи нову Database (повну сторінку, не inline) з такими властивостями:

| Назва поля     | Тип        |
| -------------- | ---------- |
| `Name`         | Title      |
| `Date`         | Date       |
| `Participants` | Text       |

> ⚠️ Назви полів мають збігатися **точно** — `Name`, `Date`, `Participants` (з великої літери).

### 3. Підключи інтеграцію до бази

1. Відкрий створену базу даних
2. Натисни `•••` у верхньому правому куті → **Connections**
3. Обери свою інтеграцію `Meet Notetaker` зі списку

### 4. Скопіюй Database ID

З URL бази даних:

```
https://www.notion.so/myworkspace/352146034ca98057b666f23bb6ae2512?v=...
                                  └──────────────┬───────────────┘
                                            Database ID
```

Database ID — це 32-значний рядок між workspace name і `?v=` у URL.

### 5. Введи креди в екстеншн

1. Клікни іконку екстеншну в Chrome
2. Встав **Notion Token** і **Database ID** у відповідні поля
3. Натисни **Save**

Креди зберігаються локально у твоєму браузері (`chrome.storage.sync`) і синхронізуються між пристроями через Google акаунт. Вони ніколи не передаються нікуди крім API Notion.

## 🚀 Як користуватись

1. Відкрий Google Meet та увімкни captions (CC) — без них транскрипт не зчитається
2. По завершенні зустрічі натисни на іконку екстеншну
3. Транскрипт автоматично збережеться у твою Notion базу

## 🗂 Структура проекту

```
meet-notetaker/
├── manifest.json     # Конфігурація екстеншну (Manifest V3)
├── background.js     # Service worker — спілкування з Notion API
├── content.js        # Зчитує транскрипт зі сторінки Meet
├── popup.html        # UI вікна налаштувань
├── popup.js          # Логіка popup — збереження кредів
├── hook.js           # Інжектиться у сторінку Meet
├── diagnose.js       # Допоміжний скрипт для відлагодження
├── styles.css        # Стилі popup
└── icons/            # Іконки екстеншну
```

## 🔒 Безпека

- Токен зберігається **тільки** у `chrome.storage.sync` твого браузера
- Жодних серверів, аналітики чи третіх сторін
- Код повністю відкритий — можеш перевірити, що нічого зайвого не передається
